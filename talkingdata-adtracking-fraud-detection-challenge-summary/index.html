这次比赛最后几个小时有人恶意公布了刚好比我高0.0001的blending kernel，遗憾的是这种行为并不违规，我到手边的银牌飞的连铜牌都不剩……

不过这次比赛收获还是很大的，最大的收获是自己的pipeline有了雏形，先整理整理放到github上

1. 比赛简介

TalkingData AdTracking Fraud Detection Challenge

比赛任务是欺诈广告点击检测。

数据包括以下特征：

  • ip
  • app（应用市场里的app id）
  • device（用户设备类型，比如iphone 6, huawei mate 7等）
  • os（设备系统）
  • channel（广告供应商的频道id）
  • click_time
  • attributed_time（如果用户点击广告之后下载了应用，这就是应用下载时间）

需要预测：

  • is_attributed（对应应用是否被下载）

4. 最终方案

  1. 使用了99%的训练数据进行训练，剩下1%是随机选出的验证集用于early stopping。使用LightGBM获得了0.9802LB，使用XGBoost获得了0.9796LB
  2. 在生成新特征的时候使用了test_supplement里的数据。使用LightGBM获得了0.9804LB，使用XGBoost获得了0.9800LB
  3. 使用十折交叉验证的方式进行训练。使用LightGBM和XGBoost都获得了0.9806LB
  4. 用随机森林作为stacker，把3中的两个模型进行stacking，获得了0.9810LB
